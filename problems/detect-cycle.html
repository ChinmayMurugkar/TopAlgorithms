Linked List Cycle

LeetCode Problem 141
Problem: Given head, the head of a linked list, determine if the linked list has a cycle in it.

Definition: A cycle exists if there is some node in the list that can be reached again by continuously following the next pointer.

Example:
Input: head = [3,2,0,-4], pos = 1
Output: true
Explanation: There is a cycle in the linked list, where the tail connects to the 1st node (0-indexed).

Solution 1 – Floyd's Cycle Finding Algorithm (Two Pointers)

Use fast and slow pointers. If there's a cycle, they will eventually meet.

public boolean hasCycle(ListNode head) { if (head == null || head.next == null) { return false; } ListNode slow = head; ListNode fast = head; while (fast != null && fast.next != null) { slow = slow.next; fast = fast.next.next; if (slow == fast) { return true; } } return false; }
Time Complexity: O(n)
Space Complexity: O(1)

Solution 2 – Hash Set Approach

Use a hash set to track visited nodes.

public boolean hasCycle(ListNode head) { Set visited = new HashSet<>(); ListNode current = head; while (current != null) { if (visited.contains(current)) { return true; } visited.add(current); current = current.next; } return false; }
Time Complexity: O(n)
Space Complexity: O(n)
Floyd's Algorithm Visualization:

List: 3 → 2 → 0 → -4 → 2 (cycle back to node 2)

Step 1: slow=3, fast=3
slow=2, fast=0

Step 2: slow=0, fast=2

Step 3: slow=-4, fast=0

Step 4: slow=2, fast=2 (they meet!)
Return true
More Examples:

Input: head = [1,2], pos = 0
Output: true
Explanation: There is a cycle where the tail connects to the 0th node.

Input: head = [1], pos = -1
Output: false
Explanation: There is no cycle in the linked list.
